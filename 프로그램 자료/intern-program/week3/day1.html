Cursor + Python AI 코딩 실습

계획서 3주차: Cursor(Agent·Tab·@참조)와 Python으로 AI 코딩 실습 — 계획서상 코딩 어시스트 IDE·완성·리팩터·디버깅에 해당합니다.

일일 스케줄
13:00–14:00
프로젝트 뼈대·가상환경
① 폴더 — `src/` `tests/` `data/sample/` `docs/`를 만들고 역할을 README 한 줄로 적습니다.
② venv — `python -m venv .venv` 후 활성화·`python --version`으로 고정합니다.
③ requirements — 필수 패키지(pandas 등)를 버전 핀으로 `requirements.txt`에 적습니다.
④ Cursor — 인터프리터를 `.venv`로 지정하고 터미널에서 동일한지 확인합니다.
⑤ git — `.gitignore`에 `.venv` `.env` `__pycache__`를 넣고 첫 커밋 메시지 규칙을 정합니다.
14:00–15:00
Tab·Agent / Composer로 첫 함수
① 과제 — CSV를 읽어 행 수·컬럼명을 반환하는 함수 시그니처를 직접 타이핑합니다.
② Tab — 본문은 Tab 제안을 받되, import와 예외 처리는 한 줄씩 검토합니다. (또는 Agent에 파일 범위를 주고 스캐폴딩 요청)
③ 실행 — `if __name__ == '__main__'` 블록에서 샘플 경로로 실행합니다.
④ 에러 — Traceback 전체를 Cursor Agent 또는 채팅에 붙여 원인→수정 패치를 요청합니다.
⑤ 커밋 — 작동 확인 후 작은 커밋으로 저장합니다.
15:00–16:00
@파일 참조 리팩터
① 분리 — 유틸·상수를 `utils.py`로 옮기고 import 경로를 정리합니다.
② @멘션 — 채팅에서 `@utils.py`를 지정해 리팩터링 범위를 제한합니다.
③ 순환 — import 순환을 끊기 위해 타입은 문자열 힌트 또는 `TYPE_CHECKING`을 씁니다.
④ 테스트 — 기존 스크립트가 동일 출력을 내는지 확인합니다.
⑤ 스타일 — black/isort/ruff 중 팀 규칙이 있으면 한 번 돌립니다.
16:00–17:00
디버깅 루틴 정착
① 재현 — 버그 입력을 최소 케이스로 줄입니다.
② 스택 — 전체 스택을 복사하고, 추측하지 말고 파일·줄 번호를 기준으로 수정합니다.
③ diff — AI 제안 패치를 적용하기 전에 diff를 읽습니다.
④ logging — `print` 대신 `logging` 한 줄을 넣어 반복 로그를 남깁니다.
⑤ 메모 — `debug_patterns_w3.md`에 막힌 유형을 적습니다.
17:00–18:00
단축키·내일(GA4) 준비
① 치트시트 — 오늘 실제로 쓴 단축키만 `cursor_shortcuts_w3.md`에 적습니다.
② 분류 — 편집·터미널·채팅·파일 탐색으로 나눕니다.
③ 내일 — GA4 또는 분석 계정 접근이 필요하면 권한·테스트 뷰를 확인합니다.
④ 백업 — 프로젝트 폴더를 zip 또는 git push로 백업합니다.
⑤ 일지 — 오늘 배운 워크플로 한 줄을 남깁니다.
