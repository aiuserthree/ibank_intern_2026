4주차 DAY 2 📅 2026-05-05 (화) 🔬 Phase 2 – AI 툴 심화·개인 역량 (3~5주)

반복 업무 자동화 — 주간 리포트 초안

계획서 4주차: 심화 활용(파일 관리·정보 보고서 생성 워크플로우)·반복 데이터(주간 리포트 초안) 자동화.

일일 스케줄
13:00–14:00
후보 업무·ROI 표
① 목록 — 반복 업무를 5개 이상 적고 각각 한 줄 설명을 붙입니다.
② 빈도 — 주 몇 회인지·월 몇 회인지 통일해 적습니다.
③ 시간 — 수동으로 한 번 할 때 분 단위로 측정합니다.
④ 난이도 — 자동화 난이도 1~5와 이유를 적습니다.
⑤ 선택 — 상위 2개를 골라 이번 주 스코프로 고정합니다.
14:00–15:00
CSV 정제 파이프라인
① pandas — `read_csv`로 읽고 `dtypes`를 확인합니다.
② 중복·결측 — 중복 키·결측 처리 규칙을 문장으로 정한 뒤 코드로 반영합니다.
③ 저장 — `cleaned.csv`와 처리 로그(행 수 전후)를 남깁니다.
④ 코드 — `auto_clean.py`에 함수 단위로 나눕니다.
⑤ 검증 — 원본 대비 행 수가 예상과 맞는지 스포트 체크합니다.
15:00–16:00
주간 리포트 워크플로우
① 트리거 — 매주 특정 요일 수동 실행 또는 스케줄을 정합니다.
② 요약 — Claude/Cowork에 넣을 프롬프트에 불릿·톤·길이를 명시합니다.
③ 저장 — Markdown으로 `reports/week_YYYY_MM_DD.md` 형태로 둡니다.
④ 연결 — `auto_report.py`에서 정제→요약→저장 순으로 호출합니다.
⑤ 플랜 B — API 실패 시 로컬 템플릿만 채우는 분기를 둡니다.
16:00–17:00
이메일 초안 자동화
① 입력 — 수신자·목적·핵심 불릿을 JSON 또는 dict로 받습니다.
② 톤 — 격식·한글 높임·서명 블록을 템플릿에 고정합니다.
③ 코드 — `email_draft.py`가 본문 문자열만 출력하게 합니다.
④ 검증 — 인사말·제목 누락을 검사하는 간단한 assert를 넣습니다.
⑤ 보안 — 실제 발송은 별도 — 초안만 파일로 저장합니다.
17:00–18:00
시간 측정·ROI 문서
① 스톱워치 — 동일 작업을 수동 vs 스크립트로 각각 2회 이상 측정합니다.
② 환산 — 주당·월당 절감 분을 표로 적습니다.
③ 파일 — `automation_roi.md`에 표·한 줄 스토리(임원용)를 넣습니다.
④ 한계 — 측정 오차·샘플 편향을 적어 과장을 피합니다.
⑤ 일지 — 오늘 배운 점을 요약합니다.
