계획서 6주차: 텍스트·데이터 파이프라인의 핵심(전처리·LLM 호출·JSON 출력)을 완성합니다.

일일 스케줄
13:00–14:00
모듈 경계
① 입력 — 허용 폴더·파일 확장자를 코드로 검증합니다.
② 출력 — 스키마(필드명·타입)를 문서와 동일하게 맞춥니다.
③ 로그 — 파일명·회전 정책을 정합니다.
④ 설정 — 상수는 `config` 한 곳에 둡니다.
⑤ 다이어그램 — 박스 화살표로 모듈 그림을 그립니다.
14:00–15:00
전처리
① 함수 — 정규화·토큰화 등을 순수 함수로 나눕니다.
② 테스트 — 샘플 문자열로 단위 테스트를 만듭니다.
③ 에러 — 사용자에게 보일 메시지를 한글로 씁니다.
④ 성능 — 대량 입력 시 메모리를 줄이는 방법을 적습니다.
⑤ 커밋 — 리팩터 커밋을 분리합니다.
15:00–16:00
LLM 호출 단계
① 배치 — 청크 크기·동시성을 정합니다.
② 타임아웃 — 초 단위로 정하고 초과 시 재시도합니다.
③ 재시도 — 지수 백오프를 적용합니다.
④ 비용 — 건당·일당 추정치를 적습니다.
⑤ 키 — `.env`만 사용하고 로그에 키가 안 남게 합니다.
16:00–17:00
검증·품질
① 샘플 — 골드 라벨 또는 수동 검수 세트를 돌립니다.
② 지표 — 정확도·완전성 등 간단 지표를 계산합니다.
③ 오류 — 유형별로 버킷에 나눕니다.
④ 개선 — 프롬프트·후처리 중 하나를 고릅니다.
⑤ 기록 — `quality_log_w6.md`.
17:00–18:00
문서·동기화
① README — 오늘 바뀐 실행 방법을 반영합니다.
② 일지 — 내일 Cowork 연동에 쓸 텍스트 샘플을 적습니다.
③ 백업 — 푸시합니다.
④ 건강 — 스트레칭.
⑤ 종료 — 긴 작업은 TODO로 남깁니다.
