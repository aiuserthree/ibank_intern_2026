AI 개발 환경 셋업 ① — Claude Cowork & Cursor

계획서 1주차: AI 개발 환경 셋업 — Claude Cowork(데스크톱 앱·지식 업무 에이전트)와 Cursor(Agent·Tab·프로젝트 규칙) 설치·계정·기본 동작 확인. (제품명·메뉴는 업데이트마다 바뀔 수 있으니 공식 문서·회사 정책을 함께 본다.)

일일 스케줄
13:00–14:00
Claude Cowork — 설치부터 첫 과제까지
① 개념 — Claude Cowork는 Claude Desktop 앱 안의 지식 업무용 에이전트입니다. 목표만 주면 로컬 폴더·파일을 읽고 정리·초안 작성까지 이어집니다. (Claude Code는 터미널·개발용 — Cowork는 GUI·문서·업무 위임에 가깝다고 구분.)
② 설치 — OS에 맞는 Claude Desktop 최신판 다운로드 → 실행 → 업데이트 적용
③ UI — 앱에서 Chat / Code / Cowork 등 모드 전환 위치 확인 (버전에 따라 표기·배치가 다를 수 있음)
④ 로그인 — Claude/Anthropic 계정 또는 회사 SSO·초대 링크 (막히면 스크린샷·에러 문구만, 비밀번호 금지)
⑤ 보안·실행 환경 — Cowork 작업이 격리된 샌드박스(로컬 VM)에서 돌아간다는 설명을 한 줄로 메모 — 회사 보안 가이드와 연결해 이해하기
⑥ 폴더·권한 — 연습 폴더 예: intern_w1_ai_setup — Cowork에 허용할 폴더만 지정, 공개 가능한 샘플만 사용
⑦ 승인 — 실행 전 계획(Plan) 확인·승인 단계가 나오면 범위·파일 접근을 읽고 진행 — 에이전트 안전의 핵심 습관
⑧ 첫 과제 — .txt 또는 짧은 .md 1개 넣고 「이 문서를 읽고 핵심만 3줄로 요약해 줘. 각 줄 끝에 근거 문장을 괄호로 표시해 줘.」— 환각이면 「모른다」로 고치게 한 번 더 요청
⑨ 선택 — MCP·데스크톱 커넥터는 회사가 허용한 것만 — 불확실하면 사용하지 않기
⑩ 기록 — 모델명·소요 시간·막힌 단계를 cowork_first_session.md에 정리
14:00–15:00
Cursor IDE — Agent·Tab·첫 코드
① 설치·계정 — cursor.com 에서 OS용 설치 → Cursor 계정 또는 GitHub 등 — 팀 정책 준수
② Agent·모델 — Agent(에이전트) 패널에서 자연어로 작업 요청 — 설정에서 회사 허용 모델만 선택 (에이전트용 Composer 등 최신 모델명은 버전별로 확인)
③ Tab 완성 — 인라인 코드 제안 — 주석 한 줄 쓴 뒤 Tab으로 수락/거절 연습
④ @ 컨텍스트 — 채팅·에이전트에 @Files, @Folder, @Codebase 등으로 참조 범위를 좁히는 방식 익히기
⑤ Project Rules — .cursor/rules 또는 팀이 쓰는 규칙 파일 위치 확인 — 일관된 스타일·금지 사항에 맞추기
⑥ Plan 모드 — (선택) 복잡한 작업을 계획 후 실행하는 흐름이 있으면 설정에서 켜고 한 번만 시험
⑦ Python — python3 --version (3.10+) — 없으면 회사 표준 경로 — python -m venv .venv 후 인터프리터 선택
⑧ 익스텐션 — Python / Pylance(또는 회사 권장) — Command Palette에서 인터프리터 지정
⑨ 에이전트·터미널 — 에이전트가 터미널·명령을 쓸 때 샌드박스 여부·허용 명령은 회사 정책 확인
⑩ 첫 파일 — hello_w1d2.py → print("ok") 실행 확인
⑪ 메모 — Cursor 앱 버전·경로·이슈를 cursor_setup_notes.md에 기록
15:00–16:00
Cowork ↔ Cursor 역할 구분 · 미니 실습
① 규칙 5줄 — (1) 긴 문서·리포트 초안·폴더 정리는 Cowork (2) 코드·리팩터·API는 Cursor Agent (3) 산출물 폴더 docs/ vs src/ … 본인 문장으로
② 샘플 텍스트 — 공개 뉴스/보도 1페이지 → sample.txt
③ Cowork — 「불릿 5개 + 한 문단 요약」→ 응답 복사
④ Cursor — sample_notes.py 또는 notes.md에 붙여넣기 → 주석으로 출처 표시 → 각 불릿에 「코드로 검증 가능?」 코멘트 (필요 시 @File로 해당 파일만 참조)
⑤ 정리 — 규칙 5줄을 한 파일로 저장
16:00–17:00

④ 스크린샷 — 이메일·토큰·내부명 가리기
⑤ 회사 데이터 — Cowork/Cursor에 넣기 전 규정 확인 — 불확실하면 공개 샘플만
