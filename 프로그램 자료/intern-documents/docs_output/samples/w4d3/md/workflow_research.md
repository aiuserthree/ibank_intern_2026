# 리서치 보고서 워크플로우 — W4D3

**날짜:** 2026-05-06 (수)
**일정:** W4D3 — 오픈클로 템플릿·워크플로우 설계
**작성자:** 인턴

---

## 워크플로우 개요

**목적:** 키워드 입력 → 자동 리서치 → 인사이트 보고서 생성

**트리거:** 수동 (주제 입력 시)

**흐름:**
1. 키워드 입력 (예: "공공기관 AI 도입 현황 2026")
2. Genspark Sparkpage로 웹 수집
3. Cowork가 수집 내용을 분석·정리
4. Markdown 보고서 생성
5. `reports/research_TOPIC_DATE.md` 저장

---

## 입력 스키마

```json
{
  "topic": "공공기관 AI 도입 현황",
  "year": 2026,
  "depth": "brief",
  "language": "ko",
  "output_sections": ["배경", "현황", "시사점", "참고 출처"]
}
```

---

## Cowork 프롬프트

```
다음 리서치 결과를 보고서 형식으로 정리하세요.

구성:
1. 배경 (2~3줄)
2. 현황 (핵심 사실 3개, 수치 포함)
3. 시사점 (2개)
4. 참고 출처 (URL 또는 매체명)

리서치 내용: {research_text}
```

---

## 품질 기준

| 항목 | 기준 |
|------|------|
| 수치 | 최소 1개 이상 포함 |
| 출처 | 최소 2개 |
| 길이 | 300~500자 |
| 언어 | 한국어 |

---

## 설계 원칙

- 각 단계는 독립적으로 테스트 가능해야 함
- 에러 처리가 모든 단계에 포함되어야 함
- 수동 개입 포인트를 명확히 표시
