## CoT란?

모델이 **단계별 추론 과정**을 먼저 출력하도록 유도해 정확도를 높이는 기법입니다.
"Let's think step by step" 또는 "단계별로 생각해줘"를 프롬프트에 넣는 것이 기본 패턴입니다.

---

## 실습 1: 단순 CoT

**입력 프롬프트:**
```
공공기관이 챗봇을 도입할 때 고려해야 할 가장 중요한 3가지 요소를
단계별로 생각하는 과정을 보여주고, 마지막에 결론을 써줘.
```

**모델 출력 (요약):**
```
[추론 1단계] 사용자 관점: 접근성·언어·응답 속도
[추론 2단계] 기관 관점: 보안·개인정보·감사 로그
[추론 3단계] 운영 관점: 유지보수·업데이트 주기·비용

결론: 공공 챗봇 도입 시 가장 중요한 요소는
(1) 개인정보 처리 정책 준수, (2) 접근성(모바일·장애인 지원), (3) 오류 응답 감지·처리 체계입니다.
```

**평가:** CoT 없을 때보다 누락 항목이 줄고 근거가 명확해짐 ✅

---

## 실습 2: Few-shot CoT

**입력 프롬프트:**
```
예시를 보고 같은 방식으로 답해줘.

예시:
질문: 이메일 요약 자동화가 유용한 직군은?
추론: 이메일 양이 많고 → 반복 분류가 필요하고 → 빠른 응답이 중요한 직군
결론: 영업·CS·행정 담당자

질문: 문서 검색 자동화가 가장 효과적인 환경은?
```

**모델 출력:**
```
추론: 문서가 많고 → 검색 쿼리가 자연어에 가깝고 → 정확한 출처가 중요한 환경
결론: 법률·규제·기술 문서가 누적된 기업·공공기관
```

**평가:** 예시 형식을 정확히 따라옴 ✅

---

## 핵심 학습 포인트

1. CoT는 **복잡한 판단**이 필요한 작업에 효과적 (단순 FAQ에는 과잉)
2. "단계별로"라는 한국어 지시어가 잘 작동
3. Few-shot CoT는 출력 형식까지 제어 가능
