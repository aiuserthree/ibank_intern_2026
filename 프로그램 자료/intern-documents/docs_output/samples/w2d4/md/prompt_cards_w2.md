## Card 01 — 문서 요약 (3줄)

**용도:** 긴 보고서·보도자료를 빠르게 파악할 때
**프롬프트:**
```
다음 문서를 읽고 핵심을 3줄로 요약해줘.
각 줄 끝에 근거 문장을 괄호로 짧게 인용해줘.
전문 용어는 쉬운 말로 풀어줘.
```
**결과 품질:** ★★★★☆
**주의:** 문서가 10,000자 이상이면 섹션별로 나눠 입력할 것

## Card 02 — 구조화된 표 만들기

**용도:** 비정형 텍스트 → 비교 표로 변환
**프롬프트:**
```
아래 내용을 읽고 [산업 / 기업명 / AI 기능 / 효과] 열로 표를 만들어줘.
없는 정보는 N/A로 표기해줘. 표는 마크다운 형식으로 출력해줘.
```
**결과 품질:** ★★★★★
**주의:** 열 이름을 명확히 지정해야 일관된 결과

---

## Card 03 — Few-shot 분류

**용도:** 텍스트를 카테고리로 분류
**프롬프트:**
```
다음 예시를 보고 텍스트를 분류해줘.

예시:
- "민원 처리 지연 불만" → 민원/불만
- "보조금 신청 방법 문의" → 정보 문의
- "담당자 태도 칭찬" → 긍정 피드백

분류할 텍스트: [텍스트 삽입]
카테고리만 출력해줘.
```
**결과 품질:** ★★★★☆
**주의:** 예시가 3개 이상일 때 정확도 향상

---

## Card 04 — Chain-of-Thought (단계별 추론)

**용도:** 복잡한 판단이 필요한 작업
**프롬프트:**
```
다음 질문에 답하기 전에, 단계별로 생각하는 과정을 먼저 써줘.
그 다음 최종 답을 '결론:' 으로 시작해 한 줄로 써줘.

질문: [질문 삽입]
```
**결과 품질:** ★★★★★
**주의:** 추론 과정에서 오류 발견 시 "이 단계가 맞는지 확인해줘"로 재검토

## Card 05 — 역할 부여 (Role Prompting)

**용도:** 특정 전문가 관점이 필요할 때
**프롬프트:**
```
당신은 공공기관 디지털 전환 전문가입니다.
다음 AI 도입 계획을 검토하고, 리스크 3가지와 개선 제안을 주세요.

계획: [계획 내용]
```
**결과 품질:** ★★★★☆
**주의:** 너무 복잡한 역할은 오히려 품질 저하 — 단순하고 명확하게
