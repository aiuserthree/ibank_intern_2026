## 3. 키워드별 딥다이브

### 3-1. RAG

- **개념:** LLM이 답변 생성 시 외부 문서(벡터 DB)를 실시간으로 참조하는 패턴
- **공공 적용:** 법령·고시·민원 FAQ를 벡터화 → 챗봇이 근거 문서 포함해 답변
- **출처:** *(오늘 찾은 논문·블로그 URL + 날짜)*
- **추가 탐색 필요:** 검색 정확도(리콜) 높이는 청킹 전략

### 3-2. AI 에이전트

- **개념:** LLM이 툴(API·파일·브라우저)을 호출해 다단계 작업을 수행
- **실습 연결:** W1D2 Cowork·Cursor Agent 체험과 동일 패턴
- **공공 적용:** 보고서 자동 생성, 데이터 수집·요약 자동화
- **출처:** *(URL + 날짜)*

### 3-3. 멀티모달

- **개념:** 텍스트 외 이미지·표·PDF를 직접 입력해 분석
- **공공 적용:** 스캔 서식 OCR + 내용 추출, 지도 이미지 포함 보고서 분석
- **출처:** *(URL + 날짜)*

### 3-4. LLM 비용·속도 최적화

- **개념:** 소형 모델(Haiku·Flash 계열) + 프롬프트 캐싱으로 API 비용 절감
- **공공 적용:** 예산 제약 기관에서 대규모 문서 처리 시 적용
- **출처:** *(URL + 날짜)*
