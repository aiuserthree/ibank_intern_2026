## Keep — 잘한 점

**1. 데모 스크립트 동결의 효과**
W7D3에 스크립트를 FROZEN 상태로 지정하고, 이후에는 수정하지 않았다. 덕분에 리허설 2회 모두 동일한 흐름으로 진행됐고, 근육 기억이 생겼다.

**2. 멘토 피드백을 슬라이드 번호에 매핑**
피드백을 "슬라이드 5번: 수치 근거 추가"처럼 번호에 연결해 기록했더니 수정 누락이 0건이었다.

**3. 플랜 B 사전 준비**
USB에 PDF + 스크린샷을 저장해두니 리허설 중 네트워크 테스트에서 실패했을 때도 당황하지 않고 넘어갈 수 있었다.

---

## Problem — 개선이 필요한 점

**1. 엣지 케이스 E-04, E-05 미완성**
W7D1에 목록을 만들었지만 특수문자·중복 ID 처리를 끝내지 못했다. PPT 완성에 시간이 집중됐다.

**2. 리허설 1회차 목소리 떨림**
혼자 연습할 때는 괜찮았지만 멘토 앞 리허설 1회차에서 첫 30초가 불안정했다. 2회차에서는 개선됐다.

**3. Git 태그 v1.0이 데이터 패키징 완료 전에 붙여짐**
v1.0 태그를 W7D2 오전에 붙였는데, 오후에 README 수정이 있었다. 태그 시점을 좀 더 신중히 해야 한다.

## Try — 8주차에 시도할 것

**1. 발표 오프닝 30초 5회 반복 연습**
첫 30초를 안정적으로 만들어 이후 흐름을 자연스럽게 이어간다.

**2. 질문 답변 시 "먼저 핵심 한 문장, 그 다음 근거" 패턴 연습**
Q7 (비용 절감 방법)에서 설명이 길어졌다. 두괄식으로 답하는 연습을 한다.
