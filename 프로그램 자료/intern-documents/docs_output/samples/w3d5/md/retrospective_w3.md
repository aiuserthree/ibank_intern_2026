## Keep — 잘한 점 (계속 유지)

**1. 작은 커밋 습관**  
기능 단위로 커밋하니 버그 발생 시 roll-back 범위가 명확했다. `git log --oneline`으로 흐름이 한눈에 보였다.

**2. @멘션 파일 범위 지정**  
Cursor 채팅에서 `@utils.py`처럼 파일을 명시하니 AI가 엉뚱한 파일을 건드리지 않았다. 리팩터링 실수가 줄었다.

**3. 실험 결과 즉시 문서화**  
Few-shot/CoT 실험 직후 `prompt_experiment_w3.md`에 결과를 기록했더니 일 마감 시 정리 시간이 절반으로 줄었다.

## Problem — 개선이 필요한 점

**1. 디버깅 시간 과다 소비**  
W3D1에서 pandas 버전 불일치로 2시간 소비했다. requirements.txt 핀 버전을 더 꼼꼼히 확인해야 한다.

**2. API 비용 추산을 후순위로 미룸**  
Self-consistency(n=5) 실험 후 예상보다 5배 비용이 나왔다. 실험 전 토큰 추산을 먼저 해야 한다.

**3. 분석 질문 정의가 늦었음**  
W3D2에서 지표를 먼저 정하지 않고 데이터를 먼저 받아 분석 방향이 중간에 바뀌었다. 비즈니스 질문 → 지표 순서를 지키자.
