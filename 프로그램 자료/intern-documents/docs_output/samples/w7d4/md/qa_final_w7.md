## 예상 Q&A 10개

### Q1. 왜 gpt-4o-mini를 선택했나요?
**A:** 비용 대비 성능이 우수합니다. 텍스트 분류 태스크에서 gpt-4o 대비 정확도는 약 3~5% 낮지만, 비용은 건당 $0.0004로 약 15배 저렴합니다. 500건 처리 기준 약 $0.20이라 인턴 프로젝트 예산에 적합했습니다.

### Q2. 분류 정확도 83%는 충분한가요?
**A:** 현재 수준은 파이로트 기준으로는 충분합니다. 실무 적용 시에는 인간 검수를 병행하는 human-in-the-loop 방식을 권장하며, 목표는 85% 이상입니다. 경계 케이스 few-shot 예시 추가로 개선 중입니다.

### Q3. 387건은 대표성이 있나요?
**A:** 2024~2026년 공공기관 AI 보도자료 기준으로 수집 가능한 공개 자료의 약 60~70%로 추정됩니다. 다만 기관 규모·분야별 불균형이 있어 해석 시 주의가 필요합니다.

### Q4. 개인정보 처리는 어떻게 했나요?
**A:** 수집 대상은 공개 보도자료로 개인정보 포함 가능성이 낮습니다. 추가로 PII 마스킹 함수를 적용해 이름·연락처 패턴을 [MASKED]로 치환했습니다.

### Q5. 실시간으로 쓸 수 있나요?
**A:** 현재는 배치 처리만 지원합니다. 실시간 처리는 스케줄러 + API 서버 추가가 필요하며 W8 이후 확장 과제로 제안할 예정입니다.

### Q6. 데이터 편향은 없나요?
**A:** 민원·행정 자동화가 33.5%로 편중되어 있습니다. 이는 실제 공공기관 AI 도입 현황을 반영하는 것으로 보이지만, 수집 소스 편향 가능성도 배제할 수 없습니다.

### Q7. 비용을 낮출 방법이 있나요?
**A:** 임베딩 기반 사전 분류로 LLM 호출 대상을 줄이거나, 캐싱을 통해 동일 텍스트 재분류를 방지하면 20~30% 절감 가능합니다.

### Q8. 재현 가능한가요?
**A:** temperature=0으로 고정하고 동일 프롬프트·모델 버전을 사용해 동일 결과를 보장합니다. README에 실행 명령어가 1줄로 정리되어 있습니다.

### Q9. 다음 단계는 무엇인가요?
**A:** 정확도 85% 달성, 시각화 대시보드, 실시간 스케줄러 추가입니다. Cowork 연동으로 보고서 자동 생성도 개선 예정입니다.

### Q10. 이 프로젝트의 실무 가치는?
**A:** 정책 담당자가 AI 도입 현황을 수동으로 조사하는 데 주당 약 8시간이 소요됩니다. 이 파이프라인으로 2분 이내 현황 파악이 가능해져 약 95% 시간 절감(추정)이 가능합니다.
