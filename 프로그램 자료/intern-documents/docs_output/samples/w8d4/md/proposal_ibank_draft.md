# 아이뱅크 제안서 초안 — W8D4

**날짜:** 2026-06-04 (목)
**일정:** W8D4 — 회사(아이뱅크) 제안서 초안 & 평가 면담 준비
**작성자:** 인턴

---

## 제안 제목: 내부 보고서 자동화 파이프라인 도입

---

## 1. 배경 & 문제

아이뱅크 내부에서는 주간·월간 보고서를 담당자가 수작업으로 취합하는 데 팀당 평균 주 4~6시간이 소요된다(멘토 인터뷰 추정). 이번 인턴십에서 유사한 문서 자동화를 파이프라인으로 구현해 45초 내 초안 생성을 확인했다.

---

## 2. 제안 내용

### 2-1. 파이로트 범위 (4주)
- 대상: 1개 팀의 주간 보고서 (현재 수작업 소요 시간 측정 후 결정)
- 도구: Claude Cowork + 오픈클로 연동
- 방식: 작업 폴더 내 데이터 파일 → Cowork 초안 → 오픈클로 저장·공유

### 2-2. 기대 효과

| 지표 | 현재 (추정) | 파이로트 목표 |
|------|-----------|------------|
| 주간 보고서 작성 시간 | 90분 | 15분 이하 |
| 수동 취합 오류율 | 알 수 없음 | 측정 시작 |
| 담당자 만족도 | 기준 없음 | 4점 이상 (5점 만점) |

### 2-3. 리스크 & 완화책

| 리스크 | 완화책 |
|--------|--------|
| 수치 환각 | [확인 필요] 태그 + 담당자 검수 1회 유지 |
| 보안 (내부 문서) | Cowork 작업 폴더 접근 권한 제한 |
| 담당자 거부감 | 자동화 = 초안 보조, 최종 결정은 사람 강조 |

---

## 3. 다음 단계

1. 팀 리드 승인 → 파이로트 팀 선정 (1주)
2. 기준 측정 (수작업 소요 시간, n=5) → 시작 전 기록
3. 4주 파이로트 실행
4. 결과 보고 → 확장 여부 결정

---

## 4. 1페이지 요약 (임원용)

| 항목 | 내용 |
|------|------|
| 문제 | 주간 보고서 수작업 4~6시간/팀/주 |
| 제안 | Cowork 자동화 파이로트 4주 |
| 기대 효과 | 작성 시간 90분 → 15분 |
| 비용 | Cowork 기존 계정 활용, 추가 비용 없음 |
| 리스크 | 수치 환각 → 검수 1회 유지로 완화 |
| 다음 단계 | 팀 리드 승인 → 파이로트 팀 선정 |
